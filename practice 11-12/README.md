# Product Manager - Учебный проект по курсу "Фронтенд и бэкенд разработка"

## 📋 О проекте

Учебное приложение для управления товарами с системой аутентификации и авторизации на основе ролей (RBAC). Проект разработан в рамках практических занятий №7-11 и демонстрирует ключевые концепции веб-разработки:

- Аутентификация и авторизация пользователей
- JWT токены (access и refresh)
- Хеширование паролей с bcrypt
- Управление доступом на основе ролей (RBAC)
- REST API с Swagger документацией
- Клиент-серверное взаимодействие

## 🚀 Функциональность

### Роли пользователей
- **Гость** - неаутентифицированный пользователь (доступ только к регистрации и входу)
- **Пользователь** - просмотр товаров
- **Продавец** - создание и редактирование товаров
- **Администратор** - полное управление товарами и пользователями

### API Маршруты

#### Аутентификация (доступно всем)
- `POST /api/auth/register` - регистрация нового пользователя
- `POST /api/auth/login` - вход в систему
- `POST /api/auth/refresh` - обновление пары токенов
- `GET /api/auth/me` - получение информации о текущем пользователе

#### Управление пользователями (только администратор)
- `GET /api/users` - получить список пользователей
- `GET /api/users/:id` - получить пользователя по ID
- `PUT /api/users/:id` - обновить информацию пользователя
- `DELETE /api/users/:id` - заблокировать пользователя

#### Управление товарами
- `POST /api/products` - создать товар (продавец, администратор)
- `GET /api/products` - получить список товаров (все авторизованные)
- `GET /api/products/:id` - получить товар по ID (все авторизованные)
- `PUT /api/products/:id` - обновить товар (продавец, администратор)
- `DELETE /api/products/:id` - удалить товар (только администратор)

## 🛠️ Установка и запуск

### Требования
- Node.js (версия 14 или выше)
- npm (менеджер пакетов)

### Пошаговая установка

1. **Клонировать репозиторий**
   ```bash
   git clone <url-репозитория>
   cd <имя-папки-проекта>
   ```

2. **Установка зависимостей сервера**
   ```bash
   cd server
   npm install
   ```

3. **Установка зависимостей клиента**
   ```bash
   cd ../client
   npm install
   ```

### Запуск приложения

#### Запуск сервера
```bash
cd server
node app.js
# или с автоматическим перезапуском при изменениях
npx nodemon app.js
```
Сервер будет доступен по адресу: http://localhost:3000

#### Запуск клиента
```bash
cd client
npm start
```
Клиент будет доступен по адресу: http://localhost:3001

### Документация API
После запуска сервера документация Swagger доступна по адресу:
http://localhost:3000/api-docs
