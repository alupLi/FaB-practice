# Отчёт по практическим занятиям №19-23

## Реализованные практические задания

### ПЗ №19: Работа с реляционными СУБД на примере PostgreSQL

**Цель:** Создать API для управления пользователями с подключением к реляционной базе данных PostgreSQL.

**Что сделано:**
- Установлена и настроена PostgreSQL (локально или через Docker)
- Установлены зависимости: `pg`, `sequelize`, `express`
- Создана модель `User` с полями: `id`, `first_name`, `last_name`, `age`, `created_at`, `updated_at`
- Реализованы CRUD-операции:
  - `POST /api/users` — создание пользователя
  - `GET /api/users` — получение списка пользователей
  - `GET /api/users/:id` — получение конкретного пользователя
  - `PATCH /api/users/:id` — обновление информации пользователя
  - `DELETE /api/users/:id` — удаление пользователя
- Данные сохраняются в PostgreSQL

### ПЗ №22: Балансировка нагрузки в веб-приложениях

**Цель:** Настроить балансировку нагрузки между несколькими backend-серверами с помощью Nginx.

**Что сделано:**
- Создано 2+ backend-сервера на Node.js (Express), каждый отвечает на `GET /` с указанием своего идентификатора
- Backend-серверы запущены на разных портах (3000, 3001, 3002)
- Настроен Nginx в роли балансировщика нагрузки
- В конфигурации Nginx использован алгоритм Round Robin для распределения запросов
- Добавлены настройки отказоустойчивости:
  - `max_fails=2` — сервер считается недоступным после 2 неудачных попыток
  - `fail_timeout=30s` — на 30 секунд сервер исключается из балансировки
  - `backup` — резервный сервер для обработки запросов при недоступности основных
- Настроен альтернативный балансировщик HAProxy
- Проверена работа балансировки и отказоустойчивости

### ПЗ №23: Контейнеризация приложений с Docker

**Цель:** Развернуть приложение с балансировкой нагрузки с помощью Docker Compose.

**Что сделано:**
- Установлен WSL и Docker Desktop на Windows
- Создан Dockerfile для backend-сервиса на Node.js (Express)
- Создан конфигурационный файл `nginx.conf` с настройками балансировки и отказоустойчивости
- Создан `docker-compose.yml`, описывающий:
  - 3 backend-сервиса (backend1, backend2, backend3) — один резервный
  - Nginx в роли балансировщика нагрузки
  - Единую сеть `app-network` для взаимодействия контейнеров
- Весь стек запущен командой `docker compose up --build`
- Проверена балансировка: при повторных запросах `curl http://localhost/` ответ приходит от разных backend-серверов
- Проверена отказоустойчивость: при остановке одного из backend-контейнеров Nginx перестаёт направлять на него запросы
- При остановке обоих основных серверов трафик переключается на резервный сервер
```
Ответ от backend-1:
{"server":"backend-1","port":3000,"message":"Response from backend server"}

Ответ от backend-2:
{"server":"backend-2","port":3000,"message":"Response from backend server"}
```
